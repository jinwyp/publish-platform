Lavavel3 Project Structure 
---
 |---- application/     该项目(应用APP)的所有我们自己的代码,包括路由,数据层,视图.大部分编码都是在这个目录下面完成
    |----config             配置文件目录
    |----controllers        控制器和路由, 包括控制器的类,
    |----language           多语言,默认包括分页和表单验证的英语界面
    |----libraries          一些简单的可以在一个类文件完成的功能的库, 如果功能很大请做成Bundle. 默认库文件start.php被Autoloader调用
    |----migrations         用来迁移数据库. 迁移的逻辑必须手动创建.或用命令行"php artisan migrate:make "创建
    |----models             数据层,Laravel提供一个可以用PHP语法使用SQL查询器叫"Fluent", 同时用Eloquent ORM 可以把表转换成对象.默认该目录文件通过start.php被Autoloader调用
    |----tasks              通过Laravel Artisan创建自己的任务.
    |----tests              Unit 单元测试 , 如果你用PHPUnit,可以使用Artisan命令行执行测试
    |----views              视图 HTML 模板 , 同时可以修改为使用 blade.php 模板引擎Blade 解析HTML
    |----bundles.php [文件] 包管理,编辑该文件管理引入的包
    |----routes.php [文件]  路由文件,包括出错502文件管理
    |----start.php [文件]   开始程序启动文件,包括auto-loading, config 目录调用文件,可以根据情况修改

 |---- bundles/        Bundles 包管理, Bundles一个包是一个Laravel的应用,可以与自己的应用分离出来成为一个包
 |---- laravel/        Laravel 框架核心文件,很少会改动,但有时会需要查看Laravel API的类和方法
 |---- public/         网站静态文件,包括首页index.php, html, css, js 图片文件等
 |---- storage/        用于文件储存的目录,例如Sessions,或缓存文件, 这个目录必须可写权限,一般不需要修改里面的文件.
 |---- vendor/         Laravel 框架会用到的文件,但一般不是Laravel开发小组而是其他贡献者开发的功能
 |---- artisan [file]  Laravel命令行功能, 通过命令行可以运行许多任务,同时可以创建你自己的任务.只要键入 "php artisan"
 |---- paths.php [file]   Laravel 框架需要定义的路径, 一般不需要修改.

命名规范

  • 文件及文件夹: 全部英文小写字母+数字或连接符"- , _ "，不可出现其他字符 如：../css/style.css, jquery.1.x.x.js
  • 文件：调用 /libs 文件需包含版本号，压缩文件需包含min关键词，其他插件则可不包含 如：/libs/jquery.1.9.1.js /libs/modernizr-1.7.min.js fileuploader.js plugins.js
  • ID: 小駝峰式命名法
    如：firstName topBoxList footerCopyright
  • Class: [减号连接符]
    如：top-item main-box box-list-item-1
  • 尽量使用语义明确的单词命名，避免 left bottom 等方位性的词语

  • 图片文件： .png (PNG-24) .jpg (压缩率8-12)

图片

  • 切图时必须合理的压缩每张图片，提高页面加载速度，如果能用1像素的就切成1像素；

  • 一般情况下，请保存为 png-8 格式，所有能保存为静态gif的图像，都应该保存为png-8格式；

  • png-24与jpg都是一种压缩图像格式，但是与jpg不同，png-24 是无损压缩，因此不会降低图像的品质（比如jpg图像锐利边缘的噪点），这也是要求效果图使用png-24格式保存的原因；

  • jpg作为一种有损压缩格式，在每次使用它压缩的时候，均会再次降低图像的品质。多次编辑同一个jpg图像，情况会变得越来越糟糕。所以一定要从设计师手中拿到无损格式的设计稿再进行工作。一般不应该为jpg格式，除非这个图像：色值远超过256色(鲜艳而华丽)，保存为索引颜色会出现明显的梯度变化（梯田），颜色抖动（点状渐变）；

  • 一般CSS背景图不使用jpg格式的图像。

使用AngularJS 之前 请先进入 "Angular的思维"

对于前端开发, 如果之前有过jQuery的经验,转型学习 AngularJS /EmberJS ,需要有哪些思维上的转变?

  • 1 不要直接设计页面上的操作DOM功能
  • 2 不要在AngularJS,过度使用jQuery
  • 3 时刻要从结构上思考问题
  • 4 Test-driven development - 总是采用TDD 测试驱动开发模式
  • 5 理论上,指令directives 并不是仅仅把jQuery打包.

AngularJS 变量 函数 方法命名规范

Angular由模板文件和js文件组成

命名禁止使用缩写,包括英文和汉语拼音缩写, 请使用英文全写命名

模板文件中的变量,全部用小写,而且不要用缩写, 如果变量是与样式有关, 例如给ngClass或ngShow使用的变量请以css前缀开头, 例如 "cssmodalshow" 
<div modal="cssmodalshow" options="cssmodalslide" ></div>
模板中的ngRepeat中的变量 也全部使用小写 
<li  ng-repeat="layout in layouts | filter:layoutfilterlisttype "></li>
模板中的事件, javascript 控制器中的事件函数全部使用驼峰命名 例如 ng-click="clickLayout()" . 事件函数的参数名称请使用全部小写命名 
<li  ng-click="clickLayout($index, layout)"  ></li>

$scope.clickayout = function(indexid, layout) {
    // some code here
};
AngularJS 的数据层 Factory和Service中 函数全部使用驼峰命名

清除浮动方案

原则上所有的浮动都需要做好清除浮动的工作，使用统一的清除浮动样式：

.clearfix:after { content:"."; display:block; height:0; clear:both; visibility:hidden;}

.clearfix { display:inline-block; }/* Safari, Chrome, Opera, FF */

.clearfix {display:block;} /* IE */

跨浏览器的inline-block

display:-moz-inline-stack; display:inline-block; vertical-align:middle; zoom:1; *display:inline;

超出自动隐藏

overflow:hidden;/*注意不要写在最后了*/ white-space:nowrap; -o-text-overflow:ellipsis; text-overflow:ellipsis;/*IE专有*/

Table表格自动换行

table-layout:fixed; word-break:break-all; word-wrap:break-word;

滤镜

filter:Alpha(opacity=10); /* IE */
-moz-opacity:0.1; /* Moz+FF */
opacity:0.1; /* 支持css3的浏览器(FF 1.5也支持) */

or

-ms-filter:"progid: DXImageTransform.Microsoft.Alpha(Opacity=100)"; filter:alpha(opacity=100); -moz-opacity:0.1; opacity:1;
